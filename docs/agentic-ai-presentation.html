The Reasoning Loop

Instead of answering in a single pass, an agentic LLM enters a loop — thinking, calling tools, reading results, and repeating until it has everything it needs.

  • 1 LLM receives the full messages[] array and a list of available tools
  • 2 It decides: respond now, or call a tool to gather more information?
  • 3 Tool results are appended to the messages stack; the loop continues
  • 4 Loop exits when the LLM produces a final assistant text response

Parallel Tool Calls

When the LLM needs multiple independent pieces of information, it batches tool calls into one assistant message — all executed simultaneously via Promise.all().

user: “What's the weather tomorrow?”
↓ LLM batches 2 independent tools
memory_mget(“user.location”) ⚡
get_current_time(“UTC”) ⚡
↓ Both results arrive together
“Istanbul, Turkey” + “2026-04-11T14:32Z”
↓ LLM calls get_weather with combined values

Context Grows Each Iteration

Every tool call and result is appended to the messages array, consuming more of the LLM's context window. The agent must complete its task before the window fills.

Agentic AI is not a one-shot call. It's a stateful loop that iterates until the task is complete or the iteration limit is hit.
