⚙

Child processes over stdio

Each MCP server runs as a subprocess. MCPServerManager spawns and communicates via JSON-RPC 2.0.

⚡

Promise.all for parallelism

When the LLM batches multiple tool calls, mcpManager.ts runs all calls concurrently via Promise.all.

🔒

Dangerous tool approval

Tools matching DANGEROUS_TOOL_PATTERNS require explicit human-in-the-loop approval before execution.

07 / 08
summary

Key Takeaways

01
Context is a growing array
Every message — system, user, tool_use, tool_result, assistant — is appended to messages[]. The LLM reads the full array each iteration.
02
Parallel batching saves iterations
Independent tool calls can be issued in a single response. Promise.all runs them concurrently — saving a full round-trip through the LLM.
03
MCP via JSON-RPC 2.0 over stdio
Each tool is a child process. The manager spawns, communicates, and multiplexes calls. No HTTP, no sockets — just structured stdio streams.
04
Results reference earlier messages
Tool args are derived from previous tool_result entries. The LLM composes inputs by reading the stack — location from [3], time from [4].
