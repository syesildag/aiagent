Core Architecture

The Agentic
Loop

Unlike a simple prompt-response model, an agentic AI enters a continuous think → act → observe cycle. The LLM decides when to call tools, inspects the results, and loops until it can respond with confidence.

S System prompt sets identity, tools & rules
1 User sends a message
2 LLM thinks — decides what tools it needs
3 Tools execute via MCP servers (JSON-RPC 2.0)
4 Results injected → repeat until done

Tool Layer

MCP Servers —
The Toolbox

Model Context Protocol servers are independent child processes that expose typed tools to the LLM. They communicate via JSON-RPC 2.0 over stdio, spawned lazily on first use.

Tool names follow the convention

mcpManager.ts — parallel execution
// Execute all tool calls concurrently
const results = await Promise.all(
  toolCalls.map(async (call) => {
    const server = this.getServer(
      call.serverName
    );
    return server.callTool({
      name: call.toolName,
      arguments: call.input
    });
  })
);

// Both tool_results arrive together
// → injected as [3] and [4] in one shot
return results.map(
  (r, i) => ({
    type: "tool_result",
    tool_call_id: toolCalls[i].id,
    content: r.content
  })
);

Memory Model

The Context
Window

The LLM has no persistent memory between calls. Its entire understanding comes from the context window — every message re-sent on each iteration.

→ System prompt is prepended every iteration (constant cost)
→ Tool results grow the context — more tools = more tokens
→ Conversation history window (default: 10 messages) controls growth
→ Prompt caching (5-min TTL) reduces cost for repeated prefixes

Summary

Key Takeaways

🔄
Iterative Loop
The LLM loops until it produces a response with no tool_use blocks. Each iteration adds to the context.
⚡
Parallel Batching
Independent tool calls are batched in one response and executed via Promise.all — saving whole iterations.
📡
JSON-RPC 2.0
MCP servers communicate via JSON-RPC over stdio. Each server is an independent child process with typed tools.
📚
Stateless LLM
The model has no memory between calls. State lives in the messages[] array, re-sent on every iteration.
🔢
Context Growth
Each tool call + result grows the context window. History windowing (default: 10) controls token cost.
🛡️
Human-in-the-Loop
Dangerous tools (matching DANGEROUS_TOOL_PATTERNS) require explicit user approval before execution.

The power of agentic AI isn't in a single inference — it's in the loop: observe, reason, act, repeat.
